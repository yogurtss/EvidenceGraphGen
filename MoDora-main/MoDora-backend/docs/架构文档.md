# MoDora Backend 重写：架构文档

## 1. 总览

重写后的后端分为两个可独立运行的子系统，但共享同一套核心库：
- **backend-core（共享库）**：领域模型 + 核心流程编排 + 端口（接口）定义
- **backend-service（服务端）**：FastAPI 提供在线 QA 接口
- **backend-lab（实验端）**：CLI 批处理、评估与绘图

依赖方向：service/lab → core → (interfaces) ← infra 实现

## 2. 分层设计

### 2.1 API/CLI 层（入口层）
职责：
- 参数解析/校验
- 调用 core 中的 application/services
- 格式化输出（HTTP response / console / files）

禁止：
- 不直接调用底层模型/embedding
- 不做检索/回答业务逻辑

### 2.2 Core Layer（核心层）

#### Domain (领域模型)
- 纯数据对象，无外部依赖。
- 示例：`Component`, `ComponentPack`, `OCRBlock`, `OcrExtractResponse`。

#### Interfaces (接口定义)
- 定义核心逻辑所需的外部能力抽象（Protocol）。
- 示例：`LLMClient` (generate_annotation), `ImageProvider` (crop_image)。

#### Services (领域服务)
- 纯业务逻辑，不依赖具体实现。
- `StructureAnalyzer`: 负责文档结构分析（OCR Block -> Component）。
- `EnrichmentService`: 负责组件信息增强（调用 LLM 接口）。

#### Application / Orchestration (应用编排)
- `preprocess.py`: 编排 Structure 和 Enrichment 服务，串联整个预处理流程。

### 2.3 Infrastructure 层（实现）
实现 `core/interfaces` 定义的协议，或提供通用基础设施。

- `infra/llm/qwen.py`: 实现 `LLMClient`，调用本地 Qwen-VL 模型。
- `infra/pdf/cropper.py`: 实现 `ImageProvider`，使用 PyMuPDF 裁剪 PDF。
- `infra/logging`: 统一的结构化日志实现，支持 request_id/run_id 上下文注入。

## 3. 核心流程与数据流

### 3.1 预处理流程 (Preprocess Pipeline)
1.  **Input**: PDF 文件路径。
2.  **OCR (Service/Infra)**: 调用 OCR 引擎（当前通过 HTTP 接口或本地调用），生成 `OcrExtractResponse` (Blocks)。
    - *优化*: 单线程顺序执行 OCR，避免多进程开销。
3.  **Structure Analysis (Core)**: `StructureAnalyzer` 分析 Blocks，识别 Header/Footer/Title，生成初始 `ComponentPack`。
4.  **Enrichment (Core)**: `EnrichmentService` 异步/多线程处理：
    - 调用 `PDFCropper` 裁剪图片/表格区域。
    - 调用 `QwenLLMClient` 生成描述信息。
5.  **Output**: 增强后的 `ComponentPack` (JSON)。

### 3.2 在线 QA (规划中)
1. 输入：file_name(s) + query + settings
2. TreeStore 加载单文档树或构造 MROOT
3. QAPipelineService 执行检索与生成
4. 输出：Prediction

## 4. 配置体系

Settings 由三部分组成：
- Paths：base_dir/cache_dir/log_dir/output_dir
- Models：qa_model、qa_answer_model 等
- Runtime：concurrency 等

加载优先级：环境变量 > 配置文件 > 默认值。

## 5. 日志系统

- **位置**: `src/modora/core/infra/logging`
- **特性**:
    - 结构化输出 (JSON/Text)
    - 上下文注入 (`request_id` for Service, `run_id` for Lab)
    - 自动轮转 (RotatingFileHandler)
- **集成**: 在 Service (`app.py`) 和 Lab (`cli.py`, `preprocess.py`) 入口处初始化。

## 6. 模型与资源管理

- **OCR**: 单例模型加载，支持单线程处理以节省显存。
- **LLM**: 适配器模式封装本地/远程调用，通过接口解耦。
